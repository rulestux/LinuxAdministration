## 110.2 Host Security
__weight 3__


### /etc/nologin
- arquivo gerado pelo superusuário que bloqueia outros logins temporariamente, podendo incluir uma mensagem de notificação do motivo para isso, a ser exibida aos usuários que tentarem se conectar.

### nologin
- binário que pode impedir o login de um usuário quando ele é definido como seu Shell padrão:
 ```$ sudo usermod -s /sbin/nologin USERNAME ```.

### inetd
- superdaemon legado para escutar conexões de rede, fornecido pelo pacote inetutils-inetd;
- '__/etc/inetd.conf__' é o arquivo de configuração, com as colunas:
	- nome do seviço;
	- tipo socket;
	- protocolo;
	- opções e flags de controle;
	- usuário que controla o processo;
	- caminho exato do processo que será iniciado;
	- argumentos;
- comentar a linha desabilita o serviço.

### xinetd
- superdaemon usado para escutar conexões de rede, feito 'inetd';
- '__/etc/xinetd.conf__' é seu arquivo de configuração junto com os arquivos eventuais em '__/etc/xinet.d/__';
- as configurações ficam em blocos iniciados por 'service' e o nome do serviço; entre chaves ficam configrações como: 'disable = no'; flags; socket_type; wait; user; server, que recebe o caminho do serviço ou daemon;
- mudar 'disable =' para 'yes', desabilita o serviço.

### Disable Unused Services
- listar status de serviços com: ```$ sudo service --status-all ```;
- desativar no Debian com: ```$ sudo update-rc.d SERVICE-NAME remove ```;
- desativar no RedHat com: ```$ sudo chkconfig SERVICE-NAME off ```;
#### para o systemd:
- listar: ```$ systemctl list-units --state active --type service ```;
- desativar: ```$ sudo systemctl disable UNIT --now ```.
